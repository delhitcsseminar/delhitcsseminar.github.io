  • Time-Space Tradeoffs for Function Inversion December 9, 2020 Venkata Koppula One-way functions - functions that can be evaluated easily on any input, but are hard to invert - are one of the fundamental building blocks of cryptography. In this talk, we will discuss the following question : given a function f, how hard/easy is it to invert y = f(x) if we're allowed unbounded preprocessing (before receiving y). More formally, we will work in the oracle setting. An adversary is first given (unbounded) oracle access to the function f to produce an S bit state. After this preprocessing, it is given f(x) (for a random x). Now, how many further oracle calls to f are necessary/sufficient to invert f(x)?

    We will discuss (a subset of) the following results:
    - Yao's lower bound [1]: For functions with domain [N], if S bits are produced during preprocessing, and T oracle calls are required in the online phase, then S.T must be at least N.
    - Hellman/Fiat-Naor's upper bound[2,3] - One can invert f(x) using T oracle calls and S bits of preprocessing advice if S^3.T is about N^3. In particular, T = S = N^{0.75} is enough to invert.
    - Is it possible to improve the lower bound? Corrigan-Gibbs, Kogan, in a recent work [4], showed that this would imply new circuit lower bounds (and hence is believed to be hard).
    - Finally, in the last part of the talk, I will discuss the scenario where the adversary is allowed to be quantum, and even the preprocessing based advice could be quantum. (I will try to ensure that no quantum background is required. In any case, this will only be the last ten/fifteen minutes). For this, I will discuss the results from two works by Nayebi et al [5] and Chung et al [6].Â
